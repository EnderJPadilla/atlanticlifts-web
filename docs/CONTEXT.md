Atlanticlifts Website
1. Executive Summary

Atlanticlifts is a premium company specialized in vertical transportation solutions, including elevator installation, modernization, preventive maintenance, corrective maintenance and escalator solutions.

The objective of this project is to design and develop a premium corporate website that conveys trust, professionalism and technical excellence.

This document is the primary source of truth for every AI assistant and every developer participating in the project.

2. Project Goals

Primary Goal

Generate qualified leads through quotation requests.

Secondary Goals

Build trust.
Showcase completed projects.
Present company services.
Improve SEO visibility.
Create a premium brand perception.

5. Architecture

The project follows a feature-based architecture with reusable UI components.

Business logic must never be mixed with presentation.

All components should be reusable whenever possible.

6. Design Philosophy

The website must not resemble a generic elevator company.

It should look closer to a premium technology company.

Design principles:

Minimalism
Large spacing
Elegant typography
High contrast
Cinematic visuals
Smooth animations
Premium interactions
7. Visual Identity

Brand values:

Trust
Innovation
Precision
Engineering
Safety
Modernity

The website must communicate these values in every section.

8. Design System

The project must use design tokens.

Avoid hardcoded colors, spacing or font sizes.

Everything should be configurable through the Design System.

12. AI Development Rules

Every AI assistant must:

Read this document before writing code.
Respect the folder architecture.
Keep components small and reusable.
Avoid duplicated code.
Prefer composition over inheritance.
Use TypeScript strictly.
Maintain accessibility.
Optimize performance before finishing a task.
Never install dependencies without justification.
13. Definition of Done

A task is complete only if:

It compiles successfully.
It passes linting.
It is responsive.
It supports dark mode.
It follows the Design System.
It meets accessibility requirements.
It includes comments only when necessary.
It is production-ready.
